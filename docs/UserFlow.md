# NexusFlow — End-to-End User Flow

This document walks the complete journey from first sign-in to a running,
optimised project. Each step names the screen and the underlying mechanism.

## 1. Sign in

- **Screen:** `app/(auth)/login.tsx`
- A split-panel SaaS login: product value on the left, a centered auth card on
  the right (email, password with reveal toggle, remember-me).
- This is a live demo build — any email creates a workspace instantly. The server
  issues a JWT (`POST /api/login`); `AuthContext` stores it and `AuthGate` routes
  the user to the dashboard.

## 2. Dashboard (command center)

- **Screen:** `app/(tabs)/dashboard.tsx`
- A time-aware greeting, the user's avatar, and six headline stats: Teams, Open
  Tasks, Members, Completion %, Sprint status, AI plans.
- **First-run empty state** explains the four-step path to value: create a
  workspace → invite the team → describe the project → let AI plan it.
- Each existing workspace shows progress and quick links into Tasks, Sprint,
  Graph and Chat.

## 3. Create a workspace (guided wizard)

- **Component:** `components/CreateTeamModal.tsx`
- A 4-step flow:
  1. **Project information** — team name, project title, description.
  2. **Your role** — Team Leader, Project Manager or Team Member, each with its
     responsibilities explained.
  3. **Team members** — add members and tag each with a skill category
     (Frontend, Backend, DevOps, Design, AI/ML, Testing). Skill data feeds
     optimal assignment later.
  4. **AI planning** — a preview of the starter backlog generated from the
     description, plus a summary of members, tasks and role.
- On finish, `useTeams.createTeam` posts the team, members and starter tasks.

## 4. Team workspace

- **Screen:** `app/team/[teamId].tsx`
- Tabs: **Overview · Tasks · Chat · Sprint · Graph · Members · Analytics · AI Rec.**
- **Overview** is the landing tab: progress, status breakdown (To do / In
  progress / Done), the team roster, and quick jumps to every other tab.

## 5. The AI workflow

- **Tab:** Chat (`components/workspace/ChatPanel.tsx`)
- The user describes the project (e.g. *"Build an AI-powered resume screening
  platform"*). NexusFlow understands the brief, generates tasks, estimates
  effort, suggests dependencies and proposes an initial sprint. Results appear as
  ordinary tasks in the Tasks tab.

## 6. Task lifecycle

1. **Created** — manually (Tasks tab → +) or generated by AI.
2. **Prioritised** — every task gets a 0–100 priority score; the list is ordered
   highest-first.
3. **Sequenced** — dependencies define execution order; cycles are blocked.
4. **Assigned** — tasks routed to members by skill fit.
5. **Worked** — status cycles To do → In progress → Done; progress bars and
   dashboard stats update live over websockets.

## 7. Sprint lifecycle

1. Open the **Sprint** tab and set a capacity (hours).
2. Run the optimiser — it selects the highest-value tasks that fit.
3. Review utilisation %, the selected set, and any capacity warnings.
4. The team executes; **Overview** and **Dashboard** reflect completion in real
   time.

## 8. DAA Insights (demo mode)

- **Screen:** `app/daa-insights.tsx`, reachable from **Profile**.
- A separate reference for mentors/demos: purpose, where-used, input, output,
  complexity and a small visualization per algorithm. Kept out of the normal
  product flow on purpose.
